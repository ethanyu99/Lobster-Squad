## Phase 3：共享上下文 + 反馈回路

> **目标**：团队成员共享知识库，支持质量反馈和迭代修正

### 共享上下文

- [ ] 实现 **Team Context** 数据结构（目标、计划、产出物、决策记录）
- [ ] 智能上下文裁剪：下游成员启动时只注入相关上下文，而非全量
- [ ] Team Context 持久化存储

### 反馈机制

- [ ] 成员间 **反馈回路**（Feedback）：Reviewer 可打回给 Coder 修改并重试
- [ ] **广播通知**（Broadcast）：关键决策通知所有团队成员
- [ ] 支持人工介入：用户可随时插入指令修正团队方向

### 前端

- [ ] 共享白板视图：实时展示团队知识库的变化
- [ ] 成果汇总页：最终产出物的统一展示和导出
